## Задача 1. Ближайший ноль

### Условие

Дан массив длины n из неотрицательных чисел. Нули — это пустые участки. Для каждого элемента найти расстояние до ближайшего нуля.

### Подход: два прохода (слева направо + справа налево)

**Почему наивный подход не подходит:**
Для каждого элемента можно было бы искать ближайший ноль перебором влево и вправо — но это O(n²), что при n до 10⁶ слишком медленно.

**Идея:**
Ближайший ноль может быть либо слева, либо справа. Разобьём задачу на две:
1. Найти расстояние до ближайшего нуля **слева**
2. Найти расстояние до ближайшего нуля **справа**
3. Взять **минимум** из двух

**Алгоритм:**

```
Проход 1 (слева направо):
  Запоминаем позицию последнего нуля (last_zero).
  Для каждого i: расстояние_слева[i] = i - last_zero
  Если нуля слева ещё не было — ставим бесконечность.

Проход 2 (справа налево):
  Запоминаем позицию последнего нуля справа (last_zero).
  Для каждого i: расстояние_справа[i] = last_zero - i
  Итоговый ответ[i] = min(расстояние_слева[i], расстояние_справа[i])
```

**Пример:**
```
Вход:  [1, 0, 3, 5, 0, 2, 1]

Проход →: [∞, 0, 1, 2, 0, 1, 2]
Проход ←: [1, 0, 1, 1, 0, 1, 2]
Минимум:  [1, 0, 1, 1, 0, 1, 2]
```

**Сложность:** O(n) по времени, O(n) по памяти.

### Решение

```python
def nearest_zero(n, a):
    distances = [float('inf')] * n

    # Проход 1: слева направо — расстояние до последнего нуля слева
    zero_pos = -float('inf')
    for i in range(n):
        if a[i] == 0:
            zero_pos = i
            distances[i] = 0
        else:
            distances[i] = min(distances[i], i - zero_pos)

    # Проход 2: справа налево — берём минимум с расстоянием до нуля справа
    zero_pos = float('inf')
    for i in range(n - 1, -1, -1):
        if a[i] == 0:
            zero_pos = i
            distances[i] = 0
        else:
            distances[i] = min(distances[i], zero_pos - i)

    return distances
```

## Задача 5. Длинное сложение и вычитание

### Условие

Даны два числа (до 1000 цифр) и операция (`+` или `-`). Вычислить результат.

### Подход: длинная арифметика

**В чём проблема:**
В большинстве языков (C++, Java) стандартные типы (`int`, `long`) вмещают максимум ~19 цифр. Число из 1000 цифр туда не поместится.

**Как решать в общем случае (C++/Java):**
Хранить число как массив цифр и реализовать сложение/вычитание «в столбик»:

```
Сложение:
  Идём от младшего разряда к старшему.
  Складываем цифры + перенос (carry).
  Если сумма ≥ 10: записываем сумма % 10, carry = 1.
  Иначе: записываем сумму, carry = 0.

Вычитание:
  Аналогично, но с заёмом (borrow).
  Если a_i - b_i - borrow < 0: прибавляем 10, borrow = 1.
```

**В Python:**
Python имеет встроенную длинную арифметику — `int` может хранить числа произвольной длины. Поэтому решение тривиально:

### Решение

```python
def long_arithmetic(a, op, b):
    if op == '+':
        result = int(a) + int(b)
    elif op == '-':
        result = int(a) - int(b)
    return str(result)
```

## Задача 9. Вырубка деревьев

### Условие

Из n деревьев в ряд (с равными промежутками) оставить m деревьев так, чтобы расстояния между соседними оставшимися были одинаковы. Сколько способов?

### Подход: перебор шага d

**Ключевое наблюдение:**
Оставшиеся m деревьев образуют **арифметическую прогрессию** — позиции через равный шаг d:

```
start, start + d, start + 2d, ..., start + (m-1)·d
```

**Что нужно перебрать:**
Для каждого допустимого шага d посчитать, сколько стартовых позиций возможно.

**Ограничение на d:**
Последнее дерево `start + (m-1)·d` не должно выходить за пределы ряда:
```
start + (m-1)·d ≤ n - 1
```
Значит `start` может быть от 0 до `n - 1 - (m-1)·d`.
Количество стартовых позиций: `n - (m-1)·d`.

**Итоговая формула:**
```
answer = Σ (n - (m-1)·d)  для d = 1, 2, ..., ⌊(n-1)/(m-1)⌋
```

**Граничные случаи:**
- `m = 0` → 1 способ (вырубить всё)
- `m = 1` → n способов (оставить любое одно дерево)

**Пример:** n=5, m=3
```
d=1: позиции (0,1,2), (1,2,3), (2,3,4) → 3 способа
d=2: позиции (0,2,4)                   → 1 способ
Итого: 4
```

**Сложность:** O(n/m) по времени.

### Решение

```python
def tree_cutting(n, m):
    ways = 0
    if m == 0:
        ways = 1
    elif m == 1:
        ways = n
    else:
        d = 1
        while (m - 1) * d < n:
            ways += n - (m - 1) * d
            d += 1
    return ways
```

## Задача 12. Эффективная быстрая сортировка (in-place)

### Условие

Отсортировать участников по: задачам (↓), штрафу (↑), логину (лексикографически ↑). Нельзя использовать O(n) дополнительной памяти.

### Подход: QuickSort с разбиением Хоара

**Почему именно QuickSort in-place:**
- MergeSort требует O(n) доп. памяти для слияния — не подходит.
- QuickSort разбивает массив «на месте», переставляя элементы внутри.

**Алгоритм:**

```
1. Выбрать опорный элемент (pivot) — например, средний.
2. Поставить два указателя: i слева, j справа.
3. Двигать i вправо, пока arr[i] < pivot.
   Двигать j влево, пока arr[j] > pivot.
4. Если i ≤ j — поменять arr[i] и arr[j] местами, сдвинуть оба.
5. Когда i > j — рекурсивно отсортировать левую и правую части.
```

**Функция сравнения:**
Задача требует нестандартный порядок. Пишем свой `compare(a, b)`:

```python
def compare(a, b):
    if a[1] != b[1]:
        return a[1] > b[1]    # больше задач → выше
    if a[2] != b[2]:
        return a[2] < b[2]    # меньше штраф → выше
    return a[0] < b[0]        # раньше по алфавиту → выше
```

**Пример разбиения:**
```
Массив: [5, 3, 8, 1, 9, 2]
Pivot = 8 (средний элемент)

i→                ←j
[5, 3, 8, 1, 9, 2]
 i=0: 5<8, двигаем →
 i=1: 3<8, двигаем →
 i=2: 8=8, стоп
 j=5: 2<8, стоп
 Меняем: [5, 3, 2, 1, 9, 8]
 ...
 Левая часть: [5, 3, 2, 1], правая: [9, 8]
```

**Сложность:** O(n log n) в среднем, O(1) доп. памяти.

### Решение

```python
def compare(a, b):
    if a[1] != b[1]:
        return a[1] > b[1]
    if a[2] != b[2]:
        return a[2] < b[2]
    return a[0] < b[0]

def partition(arr, low, high):
    pivot = arr[(low + high) // 2]
    i, j = low, high
    while i <= j:
        while compare(arr[i], pivot):
            i += 1
        while compare(pivot, arr[j]):
            j -= 1
        if i <= j:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1
            j -= 1
    return i

def quick_sort_inplace(arr, low, high):
    if low < high:
        p = partition(arr, low, high)
        quick_sort_inplace(arr, low, p - 1)
        quick_sort_inplace(arr, p, high)
```

## Задача 15. Выпуклая оболочка (периметр)

### Условие

Дан набор точек на плоскости. Найти периметр наименьшего выпуклого многоугольника, содержащего все точки.

### Подход: алгоритм «Монотонная цепь» (Andrew)

**Что такое выпуклая оболочка:**
Представьте, что вы забили гвозди в доску (точки) и натянули на них резинку. Форма, которую примет резинка — это и есть выпуклая оболочка.

**Алгоритм:**

```
1. Отсортировать точки по x (при равенстве по y).

2. Построить НИЖНЮЮ часть оболочки:
   Идём слева направо, добавляя точки.
   Если последние 3 точки делают правый поворот (или идут прямо) —
   средняя точка «внутри», удаляем её.

3. Построить ВЕРХНЮЮ часть оболочки:
   То же самое, но идём справа налево.

4. Склеить обе части.
```

**Как проверить направление поворота (векторное произведение):**

Для трёх точек O, A, B:
```
cross = (A.x - O.x) × (B.y - O.y) - (A.y - O.y) × (B.x - O.x)
```
- `cross > 0` → поворот влево (точка снаружи, оставляем)
- `cross ≤ 0` → поворот вправо или прямо (точка внутри, удаляем среднюю)

**Пример:**
```
Точки: (0,0), (1,1), (2,0), (1,3)
Сортировка: (0,0), (1,1), (1,3), (2,0)

Нижняя часть: (0,0) → (1,1) → (2,0)
  cross(0,0; 1,1; 2,0) = 1·0 - 1·1 = -1 ≤ 0 → убираем (1,1)
  Нижняя: (0,0), (2,0)

Верхняя часть: (2,0) → (1,3) → (0,0)
  Всё ОК.

Оболочка: (0,0), (2,0), (1,3)
Периметр = dist(0,0→2,0) + dist(2,0→1,3) + dist(1,3→0,0)
```

**Сложность:** O(n log n) — из-за сортировки.

### Решение

```python
import math

def cross_product(o, a, b):
    return (a[0] - o[0]) * (b[1] - o[1]) - (a[1] - o[1]) * (b[0] - o[0])

def convex_hull(points):
    points = sorted(points)
    if len(points) <= 1:
        return points

    lower = []
    for p in points:
        while len(lower) >= 2 and cross_product(lower[-2], lower[-1], p) <= 0:
            lower.pop()
        lower.append(p)

    upper = []
    for p in reversed(points):
        while len(upper) >= 2 and cross_product(upper[-2], upper[-1], p) <= 0:
            upper.pop()
        upper.append(p)

    return lower[:-1] + upper[:-1]

def convex_hull_perimeter(n, points):
    if n <= 1:
        return 0.0
    hull = convex_hull(points)
    if len(hull) <= 1:
        return 0.0
    perimeter = sum(
        math.sqrt((hull[i][0] - hull[(i+1)%len(hull)][0])**2 +
                   (hull[i][1] - hull[(i+1)%len(hull)][1])**2)
        for i in range(len(hull))
    )
    return round(perimeter, 2)
```

## Задача 16. Очень быстрая сортировка

### Условие

Дана рекуррентная последовательность: A₁ = K, A_{i+1} = (A_i × M) mod (2³²−1) mod L. Отсортировать и найти сумму нечётных по порядку элементов по модулю L.

### Подход: генерация + сортировка + суммирование

**Шаг 1 — Генерация массива:**
```python
a[0] = K
for i in range(N - 1):
    a[i+1] = (a[i] * M) % (2**32 - 1) % L
```

**Шаг 2 — Сортировка:**
Используем встроенную сортировку (Timsort, O(n log n)).

**Шаг 3 — Суммирование нечётных по порядку элементов:**
«Нечётные по порядку» = 1-й, 3-й, 5-й... = индексы 0, 2, 4... в 0-индексации.
```python
result = 0
for i in range(0, N, 2):
    result = (result + a[i]) % L
```

**Сложность:** O(n log n) — из-за сортировки.

### Решение

```python
def fast_sort_sum(n, k, m, l):
    a = [0] * n
    a[0] = k
    MOD = 0xFFFFFFFF

    for i in range(n - 1):
        a[i + 1] = ((a[i] * m) % MOD) % l

    a.sort()

    res = 0
    for i in range(0, n, 2):
        res = (res + a[i]) % l

    return res
```

## Задача 18. Музей (максимум посетителей одновременно)

### Условие

Даны интервалы посещения музея (приход-уход). Найти максимальное число посетителей одновременно.

### Подход: заметающая прямая (sweep line)

**Идея:**
Вместо того чтобы проверять каждую минуту суток (1440 минут), представим каждое посещение как два **события**:

```
Приход в 10:00  →  событие (+1) в момент 600
Уход   в 14:30  →  событие (-1) в момент 870
```

**Алгоритм:**

```
1. Преобразовать все приходы и уходы в события.
2. Отсортировать события по времени.
   При одинаковом времени: приход РАНЬШЕ ухода
   (если кто-то пришёл в 14:00 и кто-то ушёл в 14:00,
   они считаются одновременно присутствующими).
3. Идём по событиям, ведём счётчик:
   +1 при приходе, −1 при уходе.
   Максимум счётчика = ответ.
```

**Пример:**
```
Посетители: [09:00-11:00], [10:00-12:00], [10:30-11:30]

События (в минутах):
  (540, приход), (600, приход), (630, приход),
  (660, уход),  (690, уход),   (720, уход)

Проход:
  540: +1 → текущих=1, макс=1
  600: +1 → текущих=2, макс=2
  630: +1 → текущих=3, макс=3  ← ответ
  660: -1 → текущих=2
  690: -1 → текущих=1
  720: -1 → текущих=0
```

**Трюк сортировки:**
Чтобы приход обрабатывался раньше ухода при совпадении времени, кодируем тип события числом: приход = 0, уход = 1. Тогда `(время, тип)` сортируется правильно.

**Сложность:** O(n log n) — из-за сортировки.

### Решение

```python
def museum_max_visitors(n, visits):
    events = []
    for arrive_str, depart_str in visits:
        ah, am = map(int, arrive_str.split(':'))
        dh, dm = map(int, depart_str.split(':'))
        events.append((ah * 60 + am, 0))   # приход
        events.append((dh * 60 + dm, 1))   # уход

    events.sort()

    current = 0
    max_visitors = 0
    for time, event_type in events:
        if event_type == 0:
            current += 1
        else:
            current -= 1
        max_visitors = max(max_visitors, current)

    return max_visitors
```

## Задача 22. Миллиардеры

### Условие

Миллиардеры перемещаются между городами. Для каждого дня определить, какой город лидирует по суммарному капиталу. Вывести количество дней лидерства для каждого города.

### Подход: симуляция по дням с отслеживанием максимума

**Структуры данных:**
```
person_city[имя]   → в каком городе сейчас
person_wealth[имя] → капитал (не меняется)
city_wealth[город] → суммарный капитал города
```

**Алгоритм:**

```
Для каждого дня от 1 до m:
  1. Определить текущий максимальный капитал среди городов.
  2. Начислить +1 день лидерства каждому городу с максимумом.
  3. Обработать перемещения за этот день:
     - Вычесть капитал из старого города.
     - Прибавить к новому.
```

**Сложность:** O(m * k) в худшем случае, где k — число перемещений в день.

### Решение

```python
from collections import defaultdict

def billionaires(n, billionaires_data, m, movements):
    person_city = {}
    person_wealth = {}
    city_wealth = defaultdict(int)

    for name, city, wealth in billionaires_data:
        person_city[name] = city
        person_wealth[name] = wealth
        city_wealth[city] += wealth

    moves_by_day = defaultdict(list)
    for day, name, dest in movements:
        moves_by_day[day].append((name, dest))

    days_led = defaultdict(int)

    for day in range(1, m + 1):
        if city_wealth:
            max_w = max(city_wealth.values())
            if max_w > 0:
                for city, w in city_wealth.items():
                    if w == max_w:
                        days_led[city] += 1

        if day in moves_by_day:
            for name, new_city in moves_by_day[day]:
                old_city = person_city[name]
                if old_city == new_city:
                    continue
                w = person_wealth[name]
                city_wealth[old_city] -= w
                if city_wealth[old_city] == 0:
                    del city_wealth[old_city]
                city_wealth[new_city] += w
                person_city[name] = new_city

    return dict(sorted(days_led.items()))
```

## Задача 26. Очередь с приоритетами

### Условие

Реализовать структуру с операциями: добавить элемент, извлечь минимум, уменьшить ключ элемента.

### Подход: бинарная куча (min-heap)

**Что такое куча:**
Массив, организованный как дерево, где каждый родитель ≤ своих потомков.

```
Индексы:        0
              /   \
             1     2
            / \   / \
           3   4 5   6

Родитель i:      (i-1) // 2
Левый потомок:   2*i + 1
Правый потомок:  2*i + 2
```

Минимум всегда в корне (`heap[0]`).

**Три операции:**

**push (добавить):**
```
1. Добавить элемент в конец массива.
2. sift_up: пока элемент меньше родителя — менять местами.
   Элемент «всплывает» на своё место.

Пример: добавляем 2 в кучу [1, 5, 3, 7]
  [1, 5, 3, 7, 2]
   2 < 5 (родитель) → меняем
  [1, 2, 3, 7, 5]
   2 > 1 (родитель) → стоп
```

**extract_min (извлечь минимум):**
```
1. Запомнить heap[0] (это минимум).
2. Переместить последний элемент на место корня.
3. sift_down: пока элемент больше одного из потомков —
   менять с МЕНЬШИМ потомком. Элемент «тонет» на своё место.

Пример: извлекаем из [1, 2, 3, 7, 5]
  Минимум = 1. Ставим 5 в корень: [5, 2, 3, 7]
   5 > 2 (меньший потомок) → меняем
  [2, 5, 3, 7]
   5 > 7? нет. Стоп.
```

**decrease_key (уменьшить ключ):**
```
1. Найти элемент по его ID (через словарь id → индекс).
2. Обновить значение на новое (меньшее).
3. sift_up: элемент мог стать меньше родителя → всплывает.
```

**Зачем словарь id_to_idx:**
Операция `decrease_key` ссылается на элемент по номеру операции, когда он был добавлен. Без словаря пришлось бы искать элемент в куче за O(n). Словарь даёт O(1).

**Сложность:** push, extract_min, decrease_key — всё O(log n).

### Решение

```python
class PriorityQueue:
    def __init__(self):
        self.heap = []          # пары (значение, id_операции)
        self.id_to_idx = {}     # id_операции → индекс в heap

    def swap(self, i, j):
        self.heap[i], self.heap[j] = self.heap[j], self.heap[i]
        self.id_to_idx[self.heap[i][1]] = i
        self.id_to_idx[self.heap[j][1]] = j

    def sift_up(self, i):
        while i > 0:
            p = (i - 1) // 2
            if self.heap[i][0] < self.heap[p][0]:
                self.swap(i, p)
                i = p
            else:
                break

    def sift_down(self, i):
        n = len(self.heap)
        while 2 * i + 1 < n:
            left = 2 * i + 1
            right = 2 * i + 2
            j = left
            if right < n and self.heap[right][0] < self.heap[left][0]:
                j = right
            if self.heap[i][0] <= self.heap[j][0]:
                break
            self.swap(i, j)
            i = j

    def push(self, val, op_id):
        self.heap.append((val, op_id))
        idx = len(self.heap) - 1
        self.id_to_idx[op_id] = idx
        self.sift_up(idx)

    def extract_min(self):
        if not self.heap:
            return "*"
        min_val = self.heap[0][0]
        op_id = self.heap[0][1]
        del self.id_to_idx[op_id]
        last = self.heap.pop()
        if self.heap:
            self.heap[0] = last
            self.id_to_idx[last[1]] = 0
            self.sift_down(0)
        return min_val

    def decrease_key(self, op_id, new_val):
        if op_id in self.id_to_idx:
            idx = self.id_to_idx[op_id]
            self.heap[idx] = (new_val, op_id)
            self.sift_up(idx)
```

## Задача 31. Чётность (DSU с чётностью)

### Условие

Есть скрытая последовательность из 0 и 1. Серия утверждений вида: «в отрезке [l, r] чётное/нечётное число единиц». Найти номер первого ложного утверждения.

### Подход: система непересекающихся множеств (DSU) с весами

**Ключевое наблюдение — сведение к префиксным суммам:**

Пусть `S[i]` = сумма первых i элементов (по модулю 2). Тогда:
```
Сумма на отрезке [l, r] mod 2 = S[r] ⊕ S[l-1]
```
где `⊕` — XOR.

Утверждение «на [l, r] чётное число единиц» означает: `S[r] ⊕ S[l-1] = 0`, то есть `S[r]` и `S[l-1]` одной чётности.

Утверждение «нечётное» означает: `S[r] ⊕ S[l-1] = 1`, разной чётности.

**Задача сводится к:**
Нам говорят: «S[a] и S[b] одной/разной чётности». Нужно найти первое противоречие.

**Почему DSU (Union-Find):**
Каждое утверждение связывает два элемента (S[l-1] и S[r]) отношением «одинаковые» или «разные». DSU умеет эффективно группировать элементы и проверять: если два элемента уже связаны, совпадает ли новое утверждение с накопленной информацией.

**Как хранить чётность:**
Для каждого элемента храним `parity[x]` — чётность расстояния от x до корня его дерева в DSU.

```
find(x):
  Находим корень, попутно XOR-им чётности (сжатие пути).

union(a, b, p):
  Если a и b уже в одном множестве:
    Проверяем: parity[a] ⊕ parity[b] == p?
    Если нет — ПРОТИВОРЕЧИЕ.
  Иначе:
    Объединяем деревья, вычисляем чётность нового ребра.
```

**Пример:**
```
Утверждение 1: [1, 3] — even → S[0] ⊕ S[3] = 0 (одной чётности)
  → union(0, 3, 0). Объединяем.

Утверждение 2: [2, 4] — odd → S[1] ⊕ S[4] = 1 (разной чётности)
  → union(1, 4, 1). Объединяем.

Утверждение 3: [1, 4] — even → S[0] ⊕ S[4] = 0
  → find(0) и find(4). Если они уже связаны через цепочку,
     проверяем совпадает ли. Если нет — это ложь!
```

**Сложность:** O(n · α(n)) ≈ O(n) на практике, где α — обратная функция Аккермана.

### Решение

```python
class DSU:
    def __init__(self):
        self.parent = {}
        self.parity = {}

    def find(self, i):
        if i not in self.parent:
            self.parent[i] = i
            self.parity[i] = 0
            return i
        if self.parent[i] == i:
            return i
        root = self.find(self.parent[i])
        self.parity[i] ^= self.parity[self.parent[i]]
        self.parent[i] = root
        return root

    def union(self, i, j, p):
        root_i = self.find(i)
        root_j = self.find(j)
        if root_i == root_j:
            return (self.parity[i] ^ self.parity[j]) == p
        self.parent[root_i] = root_j
        self.parity[root_i] = self.parity[i] ^ self.parity[j] ^ p
        return True

def parity_check(length, queries):
    dsu = DSU()
    for idx, (left, right, parity_str) in enumerate(queries):
        p = 0 if parity_str == "even" else 1
        if not dsu.union(left - 1, right, p):
            return idx
    return len(queries)
```
